
Если требуется более гибкое хранилище чем [[SQL#Реляционные базы данных|реляционные]] бд, можно использовать системы NoSQL (нереляционные базы данных). Такие базы данных, как [[Сервер баз данных#MongoDB|MongoDB]], DocumentDB и [[Сервер баз данных#Couchbase|CouchDB]], хранят информацию в виде слабо структурированных «документов», крайне гибких и допускающих редактирование в любое время.

Узкоспециализированные базы данных нужны для:
- Улучшения производительности
- Масштабируемости
- Удобства в работе

#### Базы данных "Ключ-Значение" (Key-Value)

- Описание: Хранение данных в виде уникального ключа, указывающего на значение. База данных — это совокупность ключей.

- Хранение: Многие такие базы данных хранят данные в RAM, что обеспечивает высокую скорость, но ограничивает объём.

- Примеры использования: Кэширование, пользовательские сессии (например, корзина на сайте).

• Популярные хранилища: [[Сервер баз данных#Redis|Redis]], Memcached, [[Сервер баз данных#DynamoDB|DynamoDB]].

#### Базы данных с широкими столбцами (Wide-Column)

- Описание: По ключу хранит несколько объектов (value), похоже на реляционную БД, но без схемы, позволяя хранить разные данные.
  ![[Pasted image 20250111161057.png]]
  
- Применение: Подходят для хранения логов, данных с умных устройств или аналитических приложений (например, данные о просмотре сериалов).

• Примеры баз: [[Сервер баз данных#Cassandra|Cassandra]], Apache HBase, ClickHouse.

#### Базы данных документов (DocumentDB)

- Описание: Универсальные базы, где можно хранить документы в виде пар ключ-значение. Не структурированные данные позволяют легко удалять информацию.

- Стандарты: Поддерживают XML, YAML, JSON.

- Группировка: Документы могут группироваться в коллекции, создавая логическую иерархию.
  ![[Pasted image 20250111161639.png]]

• Примеры баз: [[Сервер баз данных#MongoDB|MongoDB]], Amazon [[Сервер баз данных#DynamoDB|DynamoDB]], CouchDB.

#### Графовые базы данных

- Описание: Содержат узлы и рёбра, которые описывают связи между объектами.
  ![[Pasted image 20250111162103.png]]

- Применение: Необходимы для алгоритмов рекомендаций, маршрутизации и т.д.

• Популярные БД: Neo4j, Dgraph.

#### Поисковые базы данных (Search-engine Database)

- Описание: Работают как базы данных документов, анализируя информацию и создавая индекс для каждого текста.

- Пример: Аналогично указателям в конце книги.

• Примеры баз: Elasticsearch, Solr, Algolia.

#### Базы данных временных рядов (Time Series Database)

- Описание: Структурированная информация, где у каждого объекта есть временной штамп.

- Применение: Используются для мониторинга загрузки сервера и других временных данных.

• Примеры баз: InfluxDB, Prometheus.

#### Многомодульные базы данных

- Описание: Базы данных, поддерживающие несколько модулей, чтобы избежать использования множества разных БД.

• Пример: [[Сервер баз данных#Redis|Redis]] поддерживает key-value, DocumentDB, GraphDB и Time Series DB.
